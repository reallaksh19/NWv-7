NWv-7 — Insight Revamp Phase/Wave Work Instruction Index
========================================================

Purpose
-------
This package contains phase-wise text work instructions for the NWv-7 Insight revamp.

Target product definition
-------------------------
Insight must become a 24-hour event-angle intelligence desk:

For important stories/events in the last 24 hours:
  1. collect reports from several sources;
  2. remove exact duplicates, same-source rewrites, and syndicated copies;
  3. keep genuinely different angles;
  4. cluster same-event stories;
  5. match events to canonical editorial Top Story anchors;
  6. classify child stories relative to the parent event;
  7. rank events and angle quality;
  8. publish a static browser-friendly digest;
  9. render the digest in the app, with browser pipeline as fallback.

Non-negotiable architecture rules
---------------------------------
1. Do not anchor Insight to the user-filtered visible Home Top Stories list.
2. Use canonical editorial Top Story anchors before read/view/localStorage suppression.
3. Do not globally relax thresholds to increase card count.
4. Use mode-specific thresholds for anchored and independent/rising stories.
5. Do not include every weak tree.
6. Keep weak trees only if anchor-matched, rising, high-score, or needed to fill minimum page quality.
7. Do not rely on a score boost alone.
8. Use two-stage selection: anchored explainers first, independent rising/developing stories second.
9. Do not keep category-based fake time slots.
10. Use rolling 24-hour snapshots from the same broad source universe.
11. Do not classify angles as story-only labels.
12. Use parent-relative angle classification.
13. Diagnostics must be machine-readable JSON, not only UI text.
