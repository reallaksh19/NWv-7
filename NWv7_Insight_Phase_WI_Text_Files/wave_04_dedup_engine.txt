Wave 04 — Dedup Engine
======================

Goal
----
Remove duplicate noise before event clustering while preserving genuinely different angles.

Dedup must separate:
  1. exact/canonical URL duplicates;
  2. same-source rewrites;
  3. syndicated copies;
  4. same-event useful new angles.

Only 1-3 are hidden as duplicates.
4 remains eligible as an angle.

Scope
-----
Allowed:
  - add standalone Python dedup package;
  - add fixture benchmark;
  - produce duplicate records and diagnostics.

Not allowed:
  - no final integration into production snapshot unless explicitly approved;
  - no ranking change;
  - no browser UI change;
  - no event clustering change.

Dedup layers
------------
Layer 1: exact/canonical URL duplicate
  - strip tracking params;
  - normalize AMP/mobile URLs;
  - normalize host/path;
  - keep richer/latest story.

Layer 2: same-source duplicate
  - same sourceGroup;
  - high title/story similarity;
  - shared facts/numbers/entities;
  - keep richer/latest story.

Layer 3: syndicated copy
  - different sourceGroup;
  - high title/story similarity;
  - shared agency signal or shared facts;
  - do not count as independent source later.

Layer 4: same-event useful angle
  - same event but new info/new angle;
  - do not dedupe here.

Duplicate record contract
-------------------------
storyId
duplicateGroupId
duplicateType:
  exact_url
  same_source_rewrite
  syndicated_copy

Dedup result contract
---------------------

  rawStoryCount
  dedupedStoryCount
  exactDuplicateCount
  sameSourceDuplicateCount
  syndicatedDuplicateCount
  dropReasons

Benchmark fixture must include
------------------------------
1. exact URL duplicate with tracking params;
2. same-source rewritten title;
3. two cross-source syndicated copies;
4. official response angle that must be retained.

Acceptance tests
----------------
Commands:
  npm run benchmark:insight-dedup
  npm run build

Pass requirements:
  [ ] exact URL duplicate detected.
  [ ] same-source rewrite detected.
  [ ] syndicated copy detected.
  [ ] official/useful angle retained.
  [ ] output.diagnostics.rawStoryCount === fixture input count.
  [ ] dedupedStoryCount < rawStoryCount.
  [ ] duplicate records include reason.
